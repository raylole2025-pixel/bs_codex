定义任务完成率，为二值变量：
$$y_m=\mathbf{1}\{R_{m,r_m^d+1}\le \epsilon_D\}$$
其中 $R_{m,r_m^d+1}$ 是任务 $m$ 到截止时的剩余数据量，$\epsilon_D$ 是一个很小的数值容差。

于是常态任务的真正完成率写成：
$$CR^{reg}=\frac{\sum_{m\in\mathcal K^{reg}} w_m y_m}{\sum_{m\in\mathcal K^{reg}} w_m}$$
## 1. 建议把阶段2正式改成这两个子模块

### 阶段2-1：常态任务基线联合规划

这是**离线**做的，在运行开始前完成。
### 阶段2-2：临机任务事件驱动插入与局部修复

这是**在线**做的，临机任务一到就处理。

这样就和你前面提出的思路完全一致了：常态任务提前联合规划，临机任务运行时插入。
## 2. 阶段2-1：常态任务基线联合规划算法

建议写成一个**联合路由-带宽分配 MILP**，目标只保留三层：

1. 最大化真正完成率
2. 最小化跨域负载不均衡
3. 最小化切换次数
### 2.1 事件分段
固定阶段1输出的跨域配置 $S^\star$ 后，构造常态任务事件集：
$$\mathcal E^{reg}(S^\star) = \{a_m,ddl_m\mid m\in\mathcal K^{reg}\} \cup \{t_l^{on},t_l^{off}\mid l\in S^\star\} \cup \{\xi_i\}$$
其中 $\xi_i$ 表示域内拓扑变化时刻。

排序后得到：
$$\tau_0<\tau_1<\cdots<\tau_R,\qquad r=[\tau_r,\tau_{r+1}),\quad \Delta t_r=\tau_{r+1}-\tau_r$$
### 2.2 容量模型

常态任务只使用分配给常态的那部分跨域容量，这和你阶段1设定保持一致：跨域边容量取 $(1-\rho)C_X$，域内边分别取 $C_A, C_B$。

因此定义：
$$c_{e,r}^{reg}=\begin{cases}C_A, & e\in E_r^A\\[4pt] C_B, & e\in E_r^B\\[4pt] (1-\rho)C_X, & e\in E_r^{AB}(S^\star)\end{cases}$$
### 2.3 候选路径
对任务 $m$ 在分段 $r$ 的每条活动跨域链路 $l=(a,b)\in E_r^{AB}(S^\star)$，构造端到端候选路径：
$$p=\Pi_{src_m\to a,r}^{A} \oplus l \oplus \Pi_{b\to dst_m,r}^{B}$$
其中两侧域内路径只保留少量候选，避免模型过大。可以采用：

- A 侧最短路 1 条 + 低重叠备选路 1 条
- B 侧最短路 1 条 + 低重叠备选路 1 条

每条跨域链路最多组合出 2～3 条端到端候选路径。
### 2.4 决策变量
对任务 $m$、路径 $p$、分段 $r$：
$$z_{m,p,r}\in\{0,1\}$$
表示分段 $r$ 是否选择路径 $p$。
$$x_{m,p,r}\ge 0$$
表示分段 $r$ 在路径 $p$ 上分配给任务 $m$ 的速率。
$$R_{m,r}\ge 0$$
表示任务 $m$ 在分段 $r$ 开始时的剩余数据量。
$$y_m\in\{0,1\}$$
表示任务 $m$ 是否在截止前真正完成。
另外定义：
$$q_r\ge 0$$
表示分段 $r$ 内跨域链路的最大归一化利用率。
$$u_{m,r}\in\{0,1\}$$
==表示任务 $m$ 在相邻分段之间是否更换了跨域链路。==
### 2.5 约束
**单任务单分段单路径**
$$\sum_{p\in\mathcal P_{m,r}(S^\star)} z_{m,p,r}\le 1$$
其中$\mathcal P_{m,r}(S^\star)$为阶段1的结果下，时间段 $r$ 中任务 $m$ 的所有路径集合；$S^\star$为阶段1的结果，即配置方案。
**速率与选路耦合**
先定义一个静态上界：
$$\bar c_{m,p,r} = \min\Big(b_m^{\max},\ \min_{e\in p} c_{e,r}^{reg}\Big)$$
再加约束：
$$0\le x_{m,p,r}\le \bar c_{m,p,r} z_{m,p,r}$$
**单分段内发送量不能超过剩余量**
有效发送时长：
$$\Delta t_{m,p,r}^{eff} = \max \left(0, \min(\tau_{r+1}, ddl_m - \delta_{p,r}^{prop+proc}) - \tau_r\right)$$
$$\sum_{p\in\mathcal P_{m,r}(S^\star)} x_{m,p,r}\Delta t_{m,p,r}^{eff} \le R_{m,r}$$
**剩余数据量更新**
$$R_{m,r_m^a}=D_m$$
其中 $r_m^a$ 为任务 $m$ 开始的分段。
$$R_{m,r+1} = R_{m,r} - \sum_{p\in\mathcal P_{m,r}(S^\star)} x_{m,p,r}\Delta t_{m,p,r}^{eff}$$
**边容量约束**
$$\sum_m \sum_{p:e\in p} x_{m,p,r} \le c_{e,r}^{reg}, \qquad \forall e,\forall r$$
**真正完成约束**
$$R_{m,r_m^{ddl}+1}\le \epsilon_D + D_m(1-y_m)$$
其中，$r_m^{ddl}$ 为任务 $m$ 截至的分段。
只要目标里最大化 $y_m$，这个约束就够了：若任务能在截止前完成，优化器会把 $y_m$ 推到 1；否则只能是 0。

**负载均衡约束**
对每条跨域链路 $l\in E_r^{AB}(S^\star)$：
$$\sum_m \sum_{p:l(p)=l} x_{m,p,r} \le q_r \cdot (1-\rho)C_X$$
**少切换约束**
若 $l_{m,r}$ 表示任务 $m$ 在分段 $r$ 选择的跨域链路，则：
$$u_{m,r}=\mathbf{1}\{l_{m,r}\neq l_{m,r-1}\}$$
在 MILP 里把它线性化即可。论文里写成这个逻辑定义就够了。
### 2.6 目标函数

这里不要加权和，直接用三层词典序。

**第一层：最大化常态任务真正完成率**
$$\max Z_1=\sum_{m\in\mathcal K^{reg}} w_m y_m$$
**第二层：在第一层最优不变前提下，最小化负载不均衡**
$$\min Z_2=\sum_r q_r\Delta t_r$$
**第三层：在前两层基本不变前提下，最小化切换**
$$\min Z_3=\sum_m\sum_r u_{m,r}$$
### 2.7 求解方式

这部分建议这样写：
- 若规划周期不长、任务量不大：直接全周期联合求解
- 若规模太大：采用**滚动窗口离线求解**

滚动窗口并不是“在线等任务来了再算”，而是在运行前对常态任务做多窗口连续规划。这样既保留联合优化，又控制求解规模。
## **3. 阶段2-2：临机任务插入算法设计**
**第一层无扰动插入；第二层受限单任务抢占；第三层尽力传输。**
### 3.1 建议采用的总表述

在阶段1确定跨域链路配置方案 $S^\star$ 后，阶段2在固定 $S^\star$ 下进行事件驱动调度。两阶段共用同一套物理规则：单任务单分段单路径、允许跨分段暂停与切换、所有边容量受限；其中阶段1仅允许常态任务占用 $(1-\rho)C_X$ 的跨域容量，为临机任务预留 $\rho C_X$。阶段2-1离线生成常态任务基线计划，阶段2-2在线处理临机任务插入。

为避免“baseline”与在线执行状态脱节，阶段2-2不再直接引用初始基线 $B_{reg}$，而是使用临机任务到达时刻 $t_e$ 的当前承诺后缀计划：
$$B^{com}(t_e^-)$$
其中包含截至 $t_e$ 时刻已经接受的常态任务与既有临机任务在未来各分段上的已承诺路由与带宽安排。下面的插入过程，都是在 $B^{com}(t_e^-)$ 的基础上进行局部修改。
### 3.2 事件驱动插入的基本定义
设临机任务为：
$$e=(src_e,dst_e,t_e,ddl_e,D_e,w_e,b_e^{\max},type_e),\qquad type_e=\mathrm{emg}.$$
若 $t_e\in(\tau_r,\tau_{r+1})$，则立即将当前分段切分为：
$$[\tau_r,\tau_{r+1})\rightarrow [\tau_r,t_e)\cup[t_e,\tau_{r+1}).$$
这一步含义是：临机任务到达后不必等到下一个时间片再处理。

为保证“是否在截止前完成”的定义严格成立，临机任务的在线规划视窗直接取到截止时刻：
$$\mathcal{T}_e=[t_e,ddl_e].$$
对应的分段集合定义为：
$$\mathcal{R}_e=\left\{r\mid [\tau_r,\tau_{r+1})\cap \mathcal{T}_e\neq\varnothing\right\}.$$
对每个 $r\in\mathcal{R}_e$，基于固定的 $S^\star$ 构造候选端到端路径集合 $\mathcal{P}_{e,r}(S^\star)$，路径构造方式与现有定义保持一致，即：
$$p=\Pi_{src_e\to a,r}^{A}\oplus l\oplus \Pi_{b\to dst_e,r}^{B}, \qquad l=(a,b)\in E_r^{AB}(S^\star).$$
阶段2仍可在两侧域内各保留 $K=2\sim 3$ 条候选路。

若对所有 $r\in\mathcal{R}_e$ 都有：
$$\mathcal{P}_{e,r}(S^\star)=\varnothing,$$
则说明在当前跨域配置下该临机任务结构不可达，直接进入第三层失败处理，无需再尝试抢占。
### 3.3 当前承诺计划下的剩余容量
设当前承诺后缀计划在链路 $\ell$、分段 $r$ 上的已占用带宽为：
$$u_{\ell,r}^{com}(t_e)= \sum_{m\in\mathcal{K}^{alive}(t_e)} \sum_{p:\ell\in p} x_{m,p,r}^{com}.$$
则当前剩余可用容量定义为：
$$c_{\ell,r}^{free}(t_e)=c_{\ell,r}^{total}-u_{\ell,r}^{com}(t_e), \qquad \forall r\in\mathcal{R}_e.$$
这个定义始终对应“此刻真正还承诺着什么”。在跨域链路上，它自然已经包含了阶段1预留给临机任务的 $\rho C_X$ 以及先前未被占满的空闲容量。
### 3.4 第一层：无扰动插入
第一层不改变任何已承诺任务，只在 $c_{\ell,r}^{free}(t_e)$ 上尝试插入临机任务 $e$。
$$R_{e,r_e^a}=D_e$$
决策变量：
$$z_{e,p,r}\in\{0,1\},\quad x_{e,p,r}\ge 0,\quad R_{e,r}\ge 0,\quad y_e\in\{0,1\}.$$
约束条件：
$$\sum_{p\in\mathcal{P}_{e,r}(S^\star)} z_{e,p,r}\le 1,\qquad \forall r\in\mathcal{R}_e$$
$$0\le x_{e,p,r}\le \bar c_{e,p,r} z_{e,p,r},\qquad \forall p,\forall r\in\mathcal{R}_e$$
其中：
$$\bar c_{e,p,r}=\min\left(b_e^{\max},\min_{\ell\in p} c_{\ell,r}^{tot}\right).$$
有效发送时长：
$$\Delta t_{e,p,r}^{eff} = \max\left( 0, \min(\tau_{r+1}, ddl_e-\delta_{p,r}^{prop+proc})-\tau_r \right).$$
状态与容量更新：
$$\sum_{p} x_{e,p,r}\Delta t_{e,p,r}^{eff}\le R_{e,r},\qquad \forall r\in\mathcal{R}_e$$
$$R_{e,r+1}=R_{e,r}-\sum_{p}x_{e,p,r}\Delta t_{e,p,r}^{eff},\qquad \forall r\in\mathcal{R}_e$$
$$\sum_{p:\ell\in p}x_{e,p,r}\le c_{\ell,r}^{free}(t_e),\qquad \forall \ell,\forall r\in\mathcal{R}_e$$
$$R_{e,r_e^{ddl}+1}\le \epsilon_D + D_e(1-y_e).$$
目标函数保留两层：
$$\max Z_1=y_e$$
$$\min Z_2=\sum_{r\in\mathcal{R}_e} q_r^e\Delta t_r$$
其中 $q_r^e$ 表示插入后分段 $r$ 内跨域链路的最大归一化利用率。
$$u_{\ell,r}^{com}(t_e) + \sum_{p:\ell\in p} x_{e,p,r} \le q_r^e c_{\ell,r}^{tot}, \qquad \forall \ell \in E_r^{AB}(S^\star), \forall r \in \mathcal{R}_e$$

如果第一层求得 $y_e=1$，则直接接受临机任务，并把求得的 $(z_{e,p,r},x_{e,p,r})$ 写回当前承诺计划 $B^{com}(t_e^-)$。
### 3.5 第二层：受限单任务抢占插入
若第一层失败，则进入第二层。第二层不做多任务联合修复，只允许对一个低优先级常态任务实施受限抢占。
#### 3.5.1 受影响任务集合
定义只在短视窗内与临机任务候选路径发生资源冲突的活跃常态任务集合：
$$\mathcal{A}_e= \left\{ m\in\mathcal{K}_{alive}^{reg}(t_e)\ \middle|\ \exists r\in\mathcal{R}_e,\exists p\in\mathcal{P}_{e,r}(S^\star),\exists p' \text{ s.t. }x_{m,p',r}^{com}>0,\ p\cap p'\neq\varnothing \right\}.$$
#### 3.5.2 抢占候选集合
定义可被抢占的候选集合（原本可按时完成且权重更小）：
$$\mathcal{C}_e= \left\{ m\in\mathcal{A}_e\ \middle|\ w_m<w_e,\ y_m^{com}=1 \right\}.$$
定义剩余裕量：
$$slack_m(t_e)=ddl_m-t_e-\frac{R_m(t_e)}{b_m^{\max}}.$$
按以下词典序排序作为抢占对象选择顺序：
1. 权重更小优先；
2. 裕量更大优先；
3. 与临机任务冲突资源更多优先。
#### 3.5.3 单任务局部抢占的定义
对任意候选任务 $m\in\mathcal{C}_e$，定义它与临机任务 $e$ 的冲突分段集合：
$$\Delta_{e,m} = \left\{ r\in\mathcal{R}_e\ \middle|\ \exists p\in\mathcal{P}_{e,r}(S^\star),\exists p' \text{ s.t. }x_{m,p',r}^{com}>0,\ p\cap p'\neq\varnothing \right\}.$$
抢占规则：若选择 $m$ 作为抢占对象，则仅在 $r\in\Delta_{e,m}$ 这些冲突分段上，按冲突分段**整体暂停**该任务的已承诺发送；在其余分段上，任务 $m$ 的计划保持不变。

抢占后的临时可用容量写为：
$$\tilde c_{\ell,r}^{free,m}(t_e)= \begin{cases} c_{\ell,r}^{free}(t_e)+\displaystyle\sum_{p:\ell\in p}x_{m,p,r}^{com}, & r\in\Delta_{e,m},\\[8pt] c_{\ell,r}^{free}(t_e), & r\notin\Delta_{e,m}. \end{cases}$$
#### 3.5.4 第二层的求解方式
按排序后的 $\mathcal{C}_e$ 依次取候选任务 $m$，每次都在临时可用容量上重新求解第一层的 MILP：
$$\tilde c_{\ell,r}^{free,m}(t_e)$$
- 若某个候选 $m^\star$ 使得 $y_e=1$，则接受临机任务，并正式把 $m^\star$ 在 $\Delta_{e,m^\star}$ 上的已承诺发送清零；
- 若所有候选都不能使 $y_e=1$，则第二层失败，进入第三层。

> **注意：** 第二层的本质不是“无损释放”，而是“受限优先级抢占”。被抢占任务 $m^\star$ 的后续完成率可能下降，这一代价被限制在一个任务、若干冲突分段的最小范围内。
### 3.6 第三层：失败处理与尽力传输

若第一层与第二层都不能实现 $y_e=1$，则记该临机任务未能按时完整插入：
$$y_e=0.$$
此时为避免继续扩大扰动范围，回退到原始未抢占容量 $c_{\ell,r}^{free}(t_e)$ 上求解一个 Best-Effort 小问题：
$$\max \sum_{r\in\mathcal{R}_e}\sum_{p\in\mathcal{P}_{e,r}(S^\star)} x_{e,p,r}\Delta t_{e,p,r}^{eff}$$
保留与第一层相同的单路径、剩余量更新和容量约束，但**不再要求**：
$$R_{e,r_e^{ddl}+1}\le \epsilon_D.$$
这一层不再追求完整按时完成，只尽量利用当前未承诺容量传一部分数据，避免临机任务被直接丢弃。
### 3.7 整理后的完整伪代码
```
Algorithm ED-SP-Insert(e, t_e, S*, B^com(t_e^-))

Input:
    e                  // new emergency task
    t_e                // arrival time of e
    S*                 // cross-domain configuration fixed by Stage 1
    B^com(t_e^-)       // current committed suffix plan before inserting e

Output:
    updated committed plan B^com(t_e^+)

1. If t_e lies inside current segment [τ_r, τ_{r+1}), split it into
       [τ_r, t_e) and [t_e, τ_{r+1})

2. Build emergency horizon
       T_e = [t_e, ddl_e]
       R_e = {r | [τ_r, τ_{r+1}) intersects T_e}

3. Generate candidate path sets P_{e,r}(S*) for all r in R_e
   If P_{e,r}(S*) = empty for all r in R_e:
       goto BestEffort

4. Compute current free capacities from committed plan:
       c_{l,r}^{free}(t_e) = c_{l,r}^{tot} - sum committed allocations

5. // Layer 1: direct insertion
   Solve DirectInsertMILP(e, R_e, P_{e,r}(S*), c_{l,r}^{free}(t_e))
   If y_e = 1:
       commit e into B^com
       return

6. // Layer 2: restricted single-task preemption
   Build affected set A_e
   Build candidate set C_e = {m in A_e | w_m < w_e and y_m^{com} = 1}
   Sort C_e by:
       (lower weight, larger slack, larger overlap)

7. For each m in C_e:
       Build conflict segment set Δ_{e,m}
       Temporarily release all committed transmissions of m on Δ_{e,m}
       Obtain temporary free capacities c̃_{l,r}^{free,m}(t_e)
       Re-solve DirectInsertMILP(e, R_e, P_{e,r}(S*), c̃_{l,r}^{free,m}(t_e))
       If y_e = 1:
            commit e
            confirm preemption of m on Δ_{e,m}
            return

8. // Layer 3: best effort
   BestEffort:
       Solve BestEffortMILP on original c_{l,r}^{free}(t_e)
       Set y_e = 0
       commit partial transmission if desired
       return
```
